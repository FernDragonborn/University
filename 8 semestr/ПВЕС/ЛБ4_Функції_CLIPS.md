# Лабораторна робота №4

## Зовнішні функції в середовищі CLIPS

### 1. Мета роботи

Вивчити особливості реалізації зовнішніх функцій у середовищі CLIPS для створення експертних систем.

---

### 2. Хід роботи

#### 2.1. Реалізація прикладу «Вибір резистора»

Створимо файл `resistor.clp`, що реалізує експертну систему для підбору резистора за законом Ома:

```
(deffacts resistors  ; База даних резисторів
  (resistor Ra 2)
  (resistor Rb 5)
  (resistor Rc 7))

(deffunction om     ; Зовнішня функція om(x, y) = y div x
  (?x ?y)
  (div ?y ?x))

(defrule input      ; Введення струму та напруги
  (initial-fact)
  =>
  (printout t crlf "Input current value: ")
  (bind ?i (read))
  (printout t "Input strait value: ")
  (bind ?u (read))
  (assert (numbers ?i ?u)))

(defrule take       ; Підбір резистора з БД
  (numbers ?i ?u)
  (resistor ?r =(om ?i ?u))
  =>
  (printout t crlf "You must take resistor " ?r "." crlf crlf)
  (reset)
  (halt))

(defrule nothing    ; Резистор не знайдено
  (numbers ?i ?u)
  (resistor ?r ~=(om ?i ?u))
  =>
  (printout t crlf "There is nothing for You in my database!" crlf crlf)
  (reset)
  (halt))
```

Пояснення структури програми:

- **БД `resistors`** — три резистори з опором 2, 5 та 7 Ом.
- **Функція `om`** — обчислює опір за законом Ома: R = U / I (цілочислове ділення).
- **Правило `input`** — зчитує з клавіатури значення струму і напруги, зберігає їх як факт `(numbers ?i ?u)`.
- **Правило `take`** — активізується, якщо обчислений опір `(om ?i ?u)` збігається з опором будь-якого резистора у БД. Оператор `=` перед викликом функції означає порівняння значення.
- **Правило `nothing`** — активізується, якщо обчислений опір НЕ збігається (`~=` — логічне заперечення порівняння).

#### 2.2. Тестування експертної системи

Завантажимо програму та протестуємо з різними вхідними даними:

**Тест 1: Резистор знайдено (R = 5 Ом)**

```
CLIPS> (clear)
CLIPS> (load resistor.clp)
.............
TRUE
CLIPS> (reset)
CLIPS> (run)

Input current value: 3
Input strait value: 15

You must take resistor Rb.
```

Перевірка: R = 15 / 3 = 5 Ом → резистор Rb (опір 5 Ом). Коректно.

**Тест 2: Резистор знайдено (R = 2 Ом)**

```
CLIPS> (run)

Input current value: 5
Input strait value: 10

You must take resistor Ra.
```

Перевірка: R = 10 / 5 = 2 Ом → резистор Ra (опір 2 Ом). Коректно.

**Тест 3: Резистор знайдено (R = 7 Ом)**

```
CLIPS> (run)

Input current value: 2
Input strait value: 14

You must take resistor Rc.
```

Перевірка: R = 14 / 2 = 7 Ом → резистор Rc (опір 7 Ом). Коректно.

**Тест 4: Резистор не знайдено**

```
CLIPS> (run)

Input current value: 3
Input strait value: 10

There is nothing for You in my database!
```

Перевірка: R = 10 / 3 = 3 (ціла частина). У БД немає резистора з опором 3 Ом. Коректно.

**Тест 5: Речові числа**

```
CLIPS> (run)

Input current value: 3.5
Input strait value: 5.44

There is nothing for You in my database!
```

Функція `div` виконує цілочислове ділення: 5.44 / 3.5 = 1 (ціла частина). Резистора з опором 1 Ом немає. Коректно.

#### 2.3. Аналіз бази знань

Файл `resistor.clp` є повноцінною базою знань, оскільки містить:

1. **Базу даних** — список резисторів із типом і опором.
2. **Знання закону Ома** — реалізоване як функція `om`.
3. **Знання про вибір** — правила `take` та `nothing`, що зіставляють обчислений опір з наявними резисторами.

Інтелект цієї БЗ можна підвищити:
- додаванням нових резисторів у БД (наприклад, Rd з опором 10 Ом)
- використанням складніших методик розрахунку опору замість простого закону Ома
- додаванням правил для вибору оптимального резистора за додатковими критеріями (потужність, допуск, ціна)

### 3. Висновки

У ході виконання лабораторної роботи вивчено механізм зовнішніх функцій у CLIPS:

1. Конструктор `deffunction` дозволяє створювати користувацькі функції з довільною кількістю параметрів. Ім'я змінної в CLIPS починається з символу `?`.

2. Зовнішні функції можуть викликатися всередині умов правил. Оператор `=` перед викликом функції означає порівняння обчисленого значення з полем факту, а `~=` — заперечення порівняння.

3. Функція `read` зчитує значення зі стандартного введення, `printout t` — виводить на стандартний вивід. Команда `halt` зупиняє виконання правил.

4. Розроблена експертна система демонструє три рівні знань: фактичні дані (резистори), процедурне знання (закон Ома) та евристичне знання (правила вибору).
